## 🚀 2. V2 기술 백로그 (Architecture Roadmap)

현 버전에선 시스템 안정성과 보안을 담보하기 위해 보수적으로 닫아두었으나, 피드백을 바탕으로 다음 Phase(V2)에서 진화시킬 핵심 설계 영역입니다.

#### A. 비정형 문서 & 이미지 스캔본 대응 (Hybrid OCR + VLM)
- 성능과 환각 최소화 방어를 위해 현재는 순수 Text-Table 형태의 제한적 PDF 파싱(`pdfplumber`)만을 허용합니다.
- V2에서는 Gemini 1.5 Pro나 타 Vision Language Model을 이용한 **이미지 스캔 기반 Layout Understanding 파이프라인**을 개척하여 하이브리드 Fallback 아키텍처를 도입할 예정입니다.

#### B. 상태 라이프사이클 명확화
- 파일럿 과정의 레지스트리 삭제 트릭 기능에 더해, 더 명시적인 시스템 이관 종료 트리거인 `cancel_requested`와 `cancelled` 상태를 도입하여 무기한(Zombie) 동작을 원천 방지하는 큐(Queue) 통제 기술을 개발합니다.

#### C. 프론트엔드 연동 강화
- 다운로드(`blob` 추출)나 데이터 테이블 렌더 등에서의 UI 로딩 스켈레톤, 폴백 처리 등 디자인 시스템을 본격 이식합니다.
