**프로젝트명**: Agentic AI Orchestration PoC  
**소속**: Google Antigravity  
**문서 버전**: v1.0-rc.1 (Agentic Planner + Deterministic Pipeline + Next.js UI Hardened)  
**작성일**: 2026-04-09  

---

## 1. 프로젝트 브리프 (Project Brief)

### 1.1 배경 (Background)
단일 대형 LLM 중심 구조는 비용 최적화와 레이턴시 측면에서 한계가 있습니다. 이를 `분해 - 위임 - 취합` 패턴으로 극복하기 위해 오케스트레이터와 서브에이전트 계층을 구축합니다. 특히 성능과 데이터 보안이라는 두 마리 토끼를 잡기 위해, **단일 Planner 파이프라인과 결정론적(Deterministic) Worker 파이프라인**으로 구조를 확립합니다.

### 1.2 목표 (Objectives)
- **Phase 5**: 더미 데이터를 활용하여 API 비율 제한(Rate Limits)에 최적화된 **Gemini-2.5-Flash** 기반의 단일 플래너 오케스트레이션 및 결정적(deterministic) Python Adapter 기반 위임 파이프라인 무결성을 입증합니다.
- **Phase 6+**: 파이프라인 무결성 검증 후, 비동기 백그라운드 큐(Celery/Redis) 시스템을 도입하여 대용량 파일 처리에서의 서버 급사 방어 및 트래픽 처리 능력을 극대화합니다.

<br>

## 2. 프로젝트 범위 (Scope)

### 2.1 In Scope (포함 대상)
- 복합 재무/엑셀 태스크 분해, 위임, 대사(Reconciliation) 취합 모듈.
- 에이전트 도구(Tool) 연동: `Excel Parser`, `Python Sandbox (Pandas)`, `File I/O`.
- `LangGraph` 기반 병렬 트레이싱 및 비용/품질 지표 수집.
- **Workflow Console UI**: `Next.js` 기반의 관리자 제어 평면 대시보드와 `FastAPI` 기반 백엔드 연동.

### 2.2 Out of Scope (제외 대상)
- 대규모 사용자 접근 권한(SSO / IAM) 분리 모델.

### 2.3 보안 범위 (Security Scope)
> [!IMPORTANT]
> - Phase 1 과정에서 API Key는 철저히 로컬 환경 변수에 격리되며, 오직 **진짜 데이터가 아닌 가상의 더미 엑셀 데이터**만을 입력값으로 사용합니다.

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 오케스트레이터 (Planner) 계층
| ID | 구분 | 세부 요구사항 명세 |
| :--- | :--- | :--- |
| **F-01** | **분해 로직** | 입력을 받아 `collect`, `normalize`, `draft`, `package` 순서로 종속성을 지닌 태스크로 분해 (`gemini-2.5-flash` 지정). |
| **F-02** | **동적 라우팅** | 각 태스크의 성격을 JSON 궤적(WorkflowEnvelope)으로 생성하여 Python 워커 파이프라인으로 전송. |
| **F-03** | **통합 에러 핸들링** | 워커가 예외를 뱉거나 환각 발생 시 `Loop Guard` 시스템이 내부 재시도. |

### 3.2 서브에이전트 (Worker Adapter) 계층
| ID | 구분 | 세부 요구사항 명세 |
| :--- | :--- | :--- |
| **F-11** | **File I/O 스냅샷** | 이전 단계의 태스크 ID를 역추적하여 이전 단계에서 생성한 물리 엑셀(`_raw.xlsx`, `_norm.xlsx`)을 순차적으로 읽어들여 무결성 보장. |
| **F-12** | **도구 연산(결정론)**| `Pandas` 기반 Python 샌드박스를 실행하여 LLM 의존 없이 빈셀(NaN) 방어 및 템플릿 매핑 수행. |
| **F-13** | **JSON 통신** | `Task Envelope` 할당을 받아 반드시 UI가 해석할 수 있는 JSON 형태의 `draft`로 결과 반환. |

## 4. 비기능 요구사항 (Non-Functional Requirements)
- **NF-01 (Low Latency)**: Python 어댑터 파이프라인 단일 노드 연산 응답 시간 **<= 5초** 이내.
- **NF-02 (Complex Latency)**: 엑셀 파일 대사 워크플로우 End-to-End P95 **<= 40초** 이내.
- **NF-03 (Cost Efficiency)**: 단일 대형 모델 전담 연산 구조 대비, LLM의 역할을 Planner 라우팅에만 국한시키고 반복 연산을 결정론적 Python 파이프라인으로 이관하여 토큰 소모 단가 **최소 80% 절감** 달성.
- **NF-04 (Quality/Accuracy)**: 정답셋 Pass Rate **>= 85%** 유지.
