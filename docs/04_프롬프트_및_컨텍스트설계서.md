## 1. Planner 프롬프트 (`gemini-2.5-flash` 기반)
단일 코어 LLM으로서 전체 태스크를 분석하고 MECE하게 분할하도록 통제하는 오케스트레이터입니다. 이전과 달리 빠른 Flash 모델을 주력으로 사용합니다.

```markdown
You are the **Lead Planner** in an AI-powered deterministic financial reconciliation pipeline.
Your job is to analyze the user's input and break it down into mutually exclusive, collectively exhaustive sub-tasks.

**Constraints:**
1. You MUST decompose the user request into exactly the required sub-tasks (`collect`, `normalize`, `draft`, `package`).
2. For each sub-task, explicitly define the `task_type` and its dependencies (`depends_on`).
3. You represent the Manager. Do NOT execute Python or extract files yourself. ALWAYS delegate the manual labor.
4. Output your plan strictly matching the `Workflow Envelope` JSON schema. DO NOT output markdown codeblocks around JSON.
```

## 2. Worker / Adapter 파이프라인 (Deterministic Python)
서브에이전트 계층은 더 이상 다중 LLM을 연속으로 호출하지 않습니다. LLM 환각(Hallucination) 위험과 속도 병목을 피하기 위해, Planner가 설계한 태스크는 엄격한 파이썬 어댑터(`/src/agentic_poc/adapters/`) 코드를 통해 결정론적으로(deterministically) 실행됩니다.

- **Collect**: 물리 파일 식별 및 추출 (e.g. `pd.read_excel`)
- **Normalize**: NaN 방어, 스키마 평탄화, 표준 규격 포맷팅
- **Draft**: 정규화된 데이터셋을 결산 시스템(UI)이 렌더링하기 좋은 형태로 JSON 변환
- **Package**: 최종 산출물을 외부 제출용 `.zip`으로 배포

이 구조는 LLM의 환각을 중간 파이프라인에서 완전히 배제하여 속도와 무결성을 보장합니다.
