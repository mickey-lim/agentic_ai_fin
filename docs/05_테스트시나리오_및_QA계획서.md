## 1. 테스트 베드 (Test Bed) 구성
벤치마크 테스트는 파이프라인의 에러가 전파되는 로직을 잡기 위해 각 에이전트를 모의 호출하는 `pytest` 자동화 테스트(총 15개 시나리오)로 구성합니다.

| 분류 | 등급 | 대상 계층 | 목적 |
| :---: | :---: | :--- | :--- |
| **A형** | 단위 (Unit) | Python Adapter | Pandas 연산, 멀티 파일 병렬 스냅샷, 빈 파일 등 블로킹 검증. 인메모리 테스트 및 파일 I/O 스모크 테스트. |
| **B형** | 보안/아키텍처 | FastAPI / DB | `workflow_source_files` M:N 조인 무결성 방어(방황하는 파일 GC 삭제 금지 규칙 보호), `max_length` 도스 방어 룰 검증 |
| **C형** | 통합 (E2E) | Planner/VLM/UI | 다중 파일 결합 통합 분석, Gemini 2.5 Flash / 이미지 영수증(JPG/PNG) VLM fallback 판독 성공률 및 로컬 SQLite 영속 상태 동기화 무결성 평가. (총 48개 테스트 케이스 누적 달성) |

## 2. 주요 QA 시나리오 (통합 테스트)

### 시나리오 C-01: 병렬 엑셀 대사 및 리포팅 (통합 E2E)
- **User Prompt**: "'1분기_부서별_지출결의서.xlsx' 파일들을 모두 순회해서 마케팅부와 관리부 총 지출액을 뽑아내 합산하고, '월간_결산_템플릿.json'의 포맷 스키마에 맞춰서 정리해줘."
- **Expected Decomposed Task (by `gemini-2.5-flash`)**:
  1. (Collect) '1분기_부서별_지출결의서.xlsx' 엑셀 IO -> `Python Adapter`
  2. (Normalize/Draft) 부서별 수치 크로스 대사 및 JSON 매핑 -> `Python Adapter`
- **Pass Criteria (성공 기준)**: 
  - RPM 초과 오류 없이, P95 타임루프 40초 이내에 완료되는가?
  - Immutable Snapshot이 유지되어 Worker 간 데이터 일관성이 확보되는가?

### 시나리오 A-03 / B-02: 악의적 언어적 주입 (Red-Teaming) 및 ReDoS 방어
- **User Prompt**: 아주 긴 문자열 (15,000자 이상) 또는 프롬프트 탈옥 (Jailbreak Markdown).
- **Expected Defense**:
  - `Pydantic` 레벨의 `max_length=15000` 필터에서 즉시 HTTP 422 튕기기.
  - 내부 로직 우회 후 환각 발생(Force_fail) 시 실패 카운트(`error_count` 3회) 기록 후 Registry 저장.
