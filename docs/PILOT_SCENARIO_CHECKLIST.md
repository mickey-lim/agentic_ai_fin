# 🧪 Pilot Scenario Checklist

본 문서는 실제 재무/회계 현업 사용자가 파일럿 과정에서 E2E(End-to-End)로 반드시 검증하고 피드백을 남겨야 할 필수 시나리오 체크리스트입니다.

## 1️⃣ [Core 시나리오] 다중 파일(Multi-File) 처리 파이프라인
- [ ] **다중 파일 순차/병렬 업로드**: 여러 장의 엑셀/CSV/PDF 증빙을 한 번에 Drag & Drop 하거나 다중 선택하여 업로드 후, 개별 목록 확인 및 삭제 액션이 정상 동작하는지 점검.
- [ ] **다중 파일 단일 워크플로우 결합**: 첨부된 2~3개의 파일이 하나의 워크플로우로 묶여 요약(Summary)과 컨텍스트가 정상 추출/통합 되는지 확인.
- [ ] **Human-In-The-Loop 대기**: `AI_보조` 또는 `사람_전담` 태스크가 포함된 컴포넌트에서 워크플로가 정상적으로 대기열(`interrupted` + `human_review`)에서 사용자의 승인을 기다리는지 확인.
- [ ] **패키지 최종 다운로드 확정**: 결재 완료 후 백엔드로부터 정확한 내용이 기재된 `.zip` 및 마크다운 리포트 패키지가 클라이언트로 다운로드되는지 확인.

## 2️⃣ [New 시나리오] 텍스트형 PDF 추출
- [ ] **스탠다드 제원 PDF 업로드**: 내부 시스템에서 산출한, 컬럼-행(Table)의 경계와 텍스트 드래그가 완벽하게 동작하는 영수증/청구서 PDF를 여러 장 섞어 업로드하여 값 추출 확인.
- [ ] **비표준 PDF 섞임 방어**: (예상 실패 시나리오) 스캔된 문서의 이미지, 흐릿하게 깨진 양식, 혹은 테이블 구조가 없는 PDF를 업로드 시 에러를 조기에 방어하거나 사용자 리뷰(Fallback) 상태로 유도하는지 확인.

## 3️⃣ [Admin 시나리오] 거버넌스 및 장애 조치
- [ ] **상태 모니터링 보드 확인**: 관리자 뷰에서 `running`, `completed`, `interrupted`, `failed` 등 개별 워크플로의 생애 주기가 실시간으로 추적되는지 확인.
- [ ] **일시 삭제 및 복구 통제**:
  - `running` 또는 중지 대상의 워크플로를 일시 삭제(`deleted` 상태 전환).
  - 해당 내역을 복구(`restore`)했을 때, 워커가 더 이상 불필요하게 `running` 좀비로 연산되지 않고 `interrupted` 상태 등으로 안전하게 제어권을 반환하는지 확인.
